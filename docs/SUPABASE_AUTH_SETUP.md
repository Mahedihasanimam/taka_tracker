# Supabase Auth Storage Setup

This app now supports cloud-backed registration/login via Supabase REST.

Create these tables in your Supabase SQL editor:

```sql
create table if not exists public.app_users (
  id bigint generated by default as identity primary key,
  name text not null,
  phone text not null unique,
  password text not null,
  created_at timestamptz not null default now()
);

create table if not exists public.app_auth_tokens (
  id bigint generated by default as identity primary key,
  user_id bigint not null references public.app_users(id) on delete cascade,
  token text not null unique,
  created_at timestamptz not null default now(),
  expires_at timestamptz not null
);

create index if not exists idx_app_auth_tokens_user_id
  on public.app_auth_tokens(user_id);

create index if not exists idx_app_auth_tokens_token
  on public.app_auth_tokens(token);
```

## Important

- If RLS is enabled on these tables, add policies that allow your app to read/write rows.
- Your project already reads Supabase config from:
  - `expo.extra.supabaseUrl`
  - `expo.extra.supabaseAnonKey`
  - fallback env vars `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_KEY`
